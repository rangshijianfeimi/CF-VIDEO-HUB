# 核心开发范式与行为约束 (Core Development Paradigm)

在处理任何代码修改、功能迭代或架构调整时，必须强制遵守以下原则。这优先于任何常规的软件工程防御性习惯：

1. **绝对无视向后兼容 (Zero Backward Compatibility)**
   - 鼓励进行破坏性变更（Breaking Changes）。
   - 在重构或修改逻辑时，不要为了兼容旧接口而保留冗余代码。如果新设计导致其他地方报错，直接去修改调用方代码以适配新逻辑，彻底清理旧逻辑。

2. **数据库结构随时变更 (Aggressive Database Refactoring)**
   - 随时可以直接增加、删除、重命名数据库表和字段，无需任何顾虑。
   - 不要为了平滑过渡而保留废弃字段，不要写复杂的向下兼容迁移逻辑。
   - 不要考虑旧数据兼容。
